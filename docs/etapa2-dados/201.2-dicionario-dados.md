# JOVI-102.2 – Dicionário de Dados: JOVI Flow
**Role:** Arquitetura de Dados
**Data:** 21/04/2026

## 1. Tabela: MIDIA
Armazena os arquivos binários e timestamps principais.

| Atributo | Tipo | Descrição | Regra de Negócio |
|----------|------|-----------|------------------|
| id | Integer (PK) | Identificador único | Gerado via timestamp (Date.now()) |
| data | LongBlob | Imagem em formato Base64 ou Blob | Compactação JPEG 0.85 para performance |
| createdAt | DateTime | Data e hora da captura | ISO 8601 string |

## 2. Tabela: METADADOS
Resultados do processamento IA e sensores.

| Atributo | Tipo | Descrição | Regra de Negócio |
|----------|------|-----------|------------------|
| id | Integer (PK) | Identificador único | Auto-incremento |
| midia_id | Integer (FK) | Relacionamento com MIDIA | Deleta em cascata |
| sharpness_score | Float | Score de nitidez (0 a 1) | Calculado via Canvas Laplacian/Sobel |
| category | String | Categoria detectada | 'Portrait', 'Night', 'Nature' ou 'Others' |
| aura_light_sim | Boolean | Indica uso do filtro visual | True se botão Aura Light estava ativo |
| orientation | String | Orientação do dispositivo | 'Portrait' ou 'Landscape' |

## 3. Estratégia de Armazenamento (IndexedDB)
- **Database Name:** `JOVIFlowDB`
- **Version:** 1
- **Storage Strategy:** Os dados são limpos automaticamente quando o armazenamento local atinge 80% da cota do navegador (LRU - Least Recently Used).
